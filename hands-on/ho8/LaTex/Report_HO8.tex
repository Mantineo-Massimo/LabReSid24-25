\section{Problem Statement}
L'ottava esercitazione (HO8) affronta il tema del calcolo parallelo e dell'ottimizzazione del carico di lavoro su architetture multi-core. L'obiettivo consiste nella ricerca dei numeri primi in un intervallo esteso ($1 - 1.000.000$) tramite l'impiego della libreria \textbf{Pthreads}. La sfida tecnica non risiede solo nella parallelizzazione algoritmica, ma nell'esplorazione della \textbf{Thread Affinity}, una tecnica avanzata che permette di vincolare i flussi di esecuzione a specifici core fisici per massimizzare il riuso della cache e minimizzare l'overhead dello scheduler di sistema.

\section{Stato dell'Arte}
L'efficienza del multithreading moderno dipende strettamente dall'interazione tra lo scheduler del sistema operativo e la topologia hardware.


\section{Metodologia ed Implementazione}
L'implementazione prevede la suddivisione dell'intervallo di ricerca e l'assegnazione deterministica dei core.

\subsection{Parallelizzazione Alternata}
Il carico viene distribuito equamente tramite una struttura \texttt{thread\_data\_t}, garantendo che ogni thread processi un sotto-intervallo unico (\textit{Static Partitioning}).
\begin{ccode}[Snippet Gestione Affinity]
cpu_set_t cpuset;
CPU_ZERO(&cpuset);
CPU_SET(data->id % num_cores, &cpuset); // Binding deterministico
pthread_setaffinity_np(pthread_self(), sizeof(cpu_set_t), &cpuset);
\end{ccode}

\subsection{Algoritmo di Verifica}
Viene utilizzato un algoritmo di test di primalità ottimizzato (fino alla radice quadrata di $N$) per ridurre la complessità computazionale di ogni singolo thread.


\subsection{Istruzioni di Esecuzione}
Compilazione con supporto ai thread (pthread):
\begin{terminal}[Build]
$ gcc prime_affinity.c -o affinity -pthread
\end{terminal}

\section{Risultati}
La validazione è stata condotta monitorando la distribuzione del carico in tempo reale.

\subsection{Validazione dell'Affinità (PSR)}
Tramite il comando \texttt{ps -eLo pid,tid,psr}, è stato verificato che il thread principale e i thread worker occupassero core distinti senza fluttuazioni.
\begin{terminal}[Monitoraggio PSR]
  PID   TID PSR CMD
20164 20165   0 ./prime_affinity (Thread 0)
20164 20166   1 ./prime_affinity (Thread 1)
\end{terminal}

\subsection{Vantaggi Prestazionali}
I test hanno confermato che forzare l'affinità riduce la varianza dei tempi di esecuzione (\textit{jitter}), rendendo il sistema più prevedibile per carichi di calcolo intensivo.

\section{Conclusioni e Sviluppi Futuri}
L'Hands-On 8 ha evidenziato come la vicinanza tra software e hardware sia determinante per le prestazioni. Il controllo dell'affinità trasforma lo scheduler da una "black box" a uno strumento deterministico.

\noindent \textbf{Sviluppi futuri:}
\begin{itemize}
    \item \textbf{Analisi Prestazionale}: Studio dell\'impatto della latenza di sistema sulla scalabilità dell\'architettura.
    \item \textbf{Ottimizzazione delle Risorse}: Refactoring del codice per minimizzare l\'uso della CPU in contesti ad alta concorrenza.
\end{itemize}
